Welcome to the commands page. All commands are explained in an extremely detailed way, so that it is easier to understand them in case of doubt. Click on each button to expand the explanation text. Each button has the command name.

Utility

Name: >>help
Arguments: n/a
Description:

Sends a Direct Message with simple bot help. Advanced help is located here (this page).

Name: >>ping
Arguments: n/a
Description:

Sends bot ping relatively to the Discord API. First try might not be accurate. Use at least twice in case of need of precision.

Name: >>info
Arguments: n/a
Description:

Sends information about the bot, such as the owner's Discord, the ping, the uptime, etc.

Name: >>say
Arguments: >>say <text>
Description:

Send desired text. For example, "">>say hello world" will make the bot send the message "hello world".

Name: >>suggest
Arguments: >>say <suggestion>
Description:

Send to the owner a report or suggestion. For example, "">>suggest >>help doesn't work!!!!!" will send the owner a DM saying ">>suggest >>help doesn't work!!!!!" in a specific format.

Name: >>calc
Arguments: >>calc <math expression>
Description:

Do a calculation. Nothing works above 25 characters. Attempt doing java code execution and my bot won't like it. Well, what can you do with 25 characters anyway?
Example: [Sender] >>calc 1 + 6
[Response] (raw) ```7```

Name: >>getperms
Arguments: >>getperms [@User] (not needed, by default sends author permissions)
Description:

Get the permissions in the guild from a given user, or by default from the author.

Name: >>embed
Arguments: >>embed <title> | [description] | [footer] | [titleUrl] (2nd+ args are not needed, requires only title to run successfully)
Description:

Creates an embed for you, with a few options. From left to right, the arguments become required after use. For example, if you want to add a footer, you must use a title and a description. For a title URL, you must use all existant arguments. The bars (|) are needed to seperate your arguments.
Example: [Sender] >>embed This is a title | This is a description | Totally not a footer | https://www.discordapp.com
[Response]
Notice: the link in the example in the title redirects to https://www.discordapp.com/, as This is a title.

Name: >>shuffle
Arguments: >>shuffle <word1> [word2] [word3] [wordN] (Only first argument necessary, though at least two are recommended, I mean, why would you shuffle 1 word?)
Description:

Shuffles given words' order. Useful for making random teams in games, or other fun stuff. 100% random using Collections.shuffle(). Don't blame me for using that method, perfomance isn't an issue for me yet.

Name: >>uptime
Arguments: n/a
Description:

Shorter >>info only with uptime information.

Fun

Name: >>cat, >>dog
Arguments: n/a for both
Description:

Sends images of these cute animals randomly. For the nerds, the used APIs are random.cat and dog.ceo, obviously respectively.

Name: >>react
Arguments: >>react <messageId> <emoteName>
Description:

Reacts to the given message ID and searches IN THE GUILD'S EMOTES ONLY an emote given in <emoteName>. In case any information is incorrect, an error will be returned and the command won't do what it's supposed to. Or do you expect it to fix your stupid mistakes?

Games

Name: >>8ball
Arguments: >>8ball [question]
Description:

8 random responses, one chosen, for a "question" you make for the mighty eight ball. Don't add arguments if it's a secret. Shhhh.

Name: >>scrabble
Arguments: n/a
Description:

Unscramble a given word, by choosing your theme. Still in development.

Other

Name: >>avatar
Arguments: >>avatar [userMention,userTag,userId]
Description:

Get avatar from a given user, or by default your own. Yeah, nothing like your own profile picture, right? Or... not.

Name: >>lyrics
Arguments: >>lyrics <songName>
Description:

Get lyrics by song name. Sourced from ABC Lyrics, using jagrosh's lyrics library. If you're a developer too, check the home page of the documentation for all of these libraries I use.
